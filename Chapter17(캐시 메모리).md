CPU의 연산 속도는 무어의 법칙을 따라 발전해왔음!

Memory Hierarchy : 저장장치를 계층적으로 배치함으로써 현실에서 발생하는 물리적 공간과 경제적 비용 문제를 해결하는 방법.
Trade-off : 속도 & 거리, 용량 & 가격

캐시 메모리 : CPU의 연산 흐름이 메모리 접근 지연으로 인해 끊기지 않게끔 설계된 SRAM 기반의 고속 버퍼.
SRAM 같은 경우에는, 전원이 공급되는 동안 데이터가 유지되는 정적 구조를 가집니다. -> DRAM 보다 속도가 훨씬 빠름.

캐시 메모리의 종류 : L1, L2, L3 -> CPU 코어와의 근접성 및 공유 방식에 따라 나누었음.
L1 캐시 : CPU 코어와 가장 가깝고, 속도가 가장 빠르다. 분리형 캐시 구조를 가진다.
L2 캐시 : L1보다 크지만, L3보다는 빠른 형태로, 균형이 잡혀있다. 코어 내부에 별개로 존재함.
L3 캐시 : 가장 용량이 크고, 여러 코어가 공유하는 구조로 설계되어 있음.
L1,L2 캐시 같은 경우, 각 코어 전용 공간인데, L3 캐시 같은 경우에는 공유 공간입니다.

캐시 히트 : CPU가 요청한 데이터가 캐시에 존재하여 즉시 처리되는 상태입니다.
캐시 미스 : 캐시에 데이터가 없어서, 느린 메모리에 접근해야 하는 상태입니다. -> 성능 하락을 일으킵니다.
캐시 적중률 : 전체 접근 대비 히트 비율을 의미합니다. 보통 8,90% 정도를 달성한다고 합니다.

참조 지역성의 원리 : 시간 지역성, 공간 지역성
시간 지역성 : 최근에 접근했던 메모리 공간에 다시 접근하려는 경향
공간 지역성 : 접근한 메모리 공간의 근처 주소에 다시 접근하려는 경향
