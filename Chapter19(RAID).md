RAID(Redundant Array of Independent Disks) : 여러 개의 물리적인 보조기억장치를 하나의 논리적인 보조기억장치로 결합하여 관리하는 방식.
-> 1. 데이터를 여러 장치에 분산함으로써, 병렬적으로 읽고 써서 I/O 대역폭을 극대화함.
-> 2. 데이터의 Redundacy의 확보를 통해 일부 장치에 에러가 발생하더라도 데이터 유실을 방지할 수 있게 됨.

RAID 0 : 데이터를 잘게 나누어, 여러 디스크에 번갈아 저장하는 기법을 이용해서 데이터를 저장.(스트라이핑 기법)
-> 여러 디스크가 동시에 데이터를 입출력하게 됨으로 인하여, 디스크 개수에 비례하여 성능이 향상하게 됨.
-> 단점 : 하나만 고장나더라도, 전체 데이터가 소실될 가능성이 있음.

RAID 1 : 데이터를 복사해서 2개 이상의 디스크에 동일하게 저장.(미러링 방식)
-> 특정 디스크가 고장났을 때, 복사본을 바로 사용 가능해서 복구하기 간단하며, 안전함.
-> 단점 : 복제를 통해서 데이터를 저장하므로 효율성이 낮음.

패리티 기술 : 수학적 연산을 통해 데이터의 정합성을 확인하고, 장애가 발생하면 원본 데이터를 제구성하는 로직. -> 손실 데이터를 복구함.
RAID 4 : 데이터는 스트라이핑하고, 별도로 디스크 하나를 패리티 저장용으로 할당함. -> 병목 현상 발생 우려.
RAID 5 : 패리티 정보를 모든 디스크에 골고루 분산해서 저장함. 
RAID 6 : 서로 다른 두 종류의 패리티를 생성하여 분산 저장함. -> 두 개의 디스크가 동시에 고장 나더라도, 데이터 복구가 가능함.

RAID 0 = 오직 입출력 속도가 빠른 것이 목적인 경우
RAID 1, 6 = 데이터의 안전한 보관이 목적인 경우
RAID 5 = 적당하게, 그냥 그저 그런 경우
