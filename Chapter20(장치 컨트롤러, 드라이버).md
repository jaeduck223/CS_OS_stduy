입출력장치의 애로사항 : 종류가 너무 다양하고, 데이터 전송 형식과 인터페이스의 종류 또한 다양함.
                      대부분의 입출력장치는 전송률이 매우 낮음.-> 유휴 시간의 발생. 비효율적

장치 컨트롤러 : 통신 중계, 오류 검출, 데이터 버퍼링을 통해 입출력장치의 애로사항을 극복한 하드웨어. 레지스터로 구성됨.
버퍼 : 데이터가 임시로 저장되는 공간. 데이터의 전송량 조절을 통해 시스템 전체의 흐름을 조절함.
장치 레지스터의 종류 : 데이터, 상태, 제어 레지스터

데이터 레지스터 : CPU와 장치 사이에서 주고 받는 데이터가 담기는 통로 -> 버퍼 역할의 수행. 
상태 레지스터 : 장치의 현재 상태를 저장 -> 장치 컨트롤러의 오류 검출 결과가 저장되는 곳.
제어 레지스터 : CPU가 장치에 내리는 구체적인 동작 명령을 저장 -> 장치에 특정 작업을 지시할 때 사용하는 입력 통로

CPU와 장치 컨트롤러의 협력 : 상태 확인 -> 명령 전달 -> 데이터 교환

장치 드라이버 : 장치 컨트롤러를 제어하고, 운영체제와 연결하는 소프트웨어 계층.
-> 지시 사항을 보유하고 있는 프로그램.
-> 하드웨어와 운영체제 사이를 연결함.
-> 장치 연결 시 불량의 주요 원인.
