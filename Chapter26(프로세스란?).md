프로세스 : 프로그램이 메모리에 쌓여 실행을 시작할 때, 생성하는 부산물. -> 테스크와 통하는 말.
PCB(프로세스 제어 블록) : 프로세스의 식별 및 관리를 위해 커널 영역에 생성하는 자료구조. 운영체제가 생성하며, 프로세스 생성 시 태어나고, 종료되면 사망.
PCB는 PID, 레지스터 값, 프로세스의 상태, CPU 스케줄링 정보, 메모리 정보, 프로세스의 점유 리소스 목록으로 구성합니다.
특수 프로세스 : 사용자와의 상호작용 없이 백그라운드에서 묵묵히 정해진 작업만 수행하는 프로세스 (데몬, 서비스)

시분할 방식 및 동시 실행 : 각 프로세스에 아주 짧은 시간만 할당하여 한정된 CPU 자원을 효율적으로 사용하는 기법.
문맥(Context) : 프로세스가 CPU를 반납하고 나중에 재개될 때, 이전에 어디까지 실행했는지에 대한 중간 정보
타이머 인터럽트(Timer Interrupt / Time-out Interrupt) : 문맥의 교환을 일으키는 하드웨어의 작용
인터럽트 발생 - 문맥의 획득 및 백업 - PCB 저장 - 새 문맥의 복구 - 실행의 재개

운영체제의 영역 : 정적 할당 영역, 동적 할당 영역
정적 할당 영역 : 코드 영역(CPU가 실행할 실제 기계어 명령어)과 데이터 영역(프로그램 실행 전반에 걸쳐 유지되는 전역 변수)
동적 할당 영역 : 힙 영역(프로그래머가 직접 메모리 공간을 할당하는 영역)과 스택 영역(함수 호출 시 생성되는 지역 변수나 매개 변수와 같은 일시적인 데이터)
