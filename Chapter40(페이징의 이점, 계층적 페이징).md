쓰기 시 복사 (Copy-on-Write) 전통적인 fork() 시스템 호출은 부모 프로세스의 전체 내용을 다른 메모리 공간에 통째로 복제하여 자식 프로세스를 생성하므로 시간과 메모리가 불필요하게 낭비됩니다
페이징을 활용한 쓰기 시 복사 기법은 이 문제를 다음과 같이 해결합니다.

초기 상태: 자식 프로세스가 생성되면 통째로 복제하는 대신, 일단 부모 프로세스와 동일한 프레임(메모리 공간)을 가리키게 하여 메모리를 공유합니다
쓰기 작업 발생 시: 프로세스 간에는 기본적으로 자원을 공유하지 않아야 하므로, 부모나 자식 중 한쪽이 특정 페이지에 데이터를 쓰려 할 때(수정할 때) 해당 페이지만 별도의 공간으로 복제하여 새 프레임에 할당합니다

효과: 메모리 중복 적재를 최소화하여 메모리를 훨씬 효율적으로 사용할 수 있으며, 자원 독립성도 유지할 수 있습니다

계층적 페이징 (Hierarchical Paging) 프로세스의 크기가 커지면 페이지 테이블의 크기도 덩달아 매우 커지게 되는데, 이 거대한 페이지 테이블의 모든 내용을 메모리에 올려두는 것은 심각한 메모리 낭비입니다
이를 방지하기 위해 페이지 테이블 자체를 여러 단계로 다시 페이징하여 관리하는 기법입니다
작동 방식: 기존 페이지 테이블을 여러 페이지로 쪼개고, 이를 가리키는 또 다른 '외부(Outer) 페이지 테이블'을 둡니다
이렇게 하면 CPU와 가장 가까운 최상단의 외부 페이지 테이블만 메모리에 항상 유지하고, 나머지 하위 페이지 테이블들은 당장 필요한 것들만 메모리에 적재하여 메모리를 절약할 수 있습니다

논리 주소 변환: 계층적 페이징 시스템에서는 논리 주소가 단순한 페이지 번호가 아닌, 외부 페이지 번호(p1) + 내부 페이지 번호(p2) + 변위 형태로 쪼개져 구성됩니다
다단계 구성의 한계: 2단계를 넘어 3단계, 4단계 이상으로 계층을 깊게 구성할 수도 있습니다
하지만 계층이 많아질수록 특정 페이지를 찾기 위해 거쳐야 할 과정이 늘어나고, 특히 페이지 폴트 발생 시 메모리를 참조해야 하는 횟수가 그만큼 증가하므로 무조건 단계가 많다고 성능이 좋은 것은 아닙니다
