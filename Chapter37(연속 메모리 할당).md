스와핑 (Swapping) : 당장 사용하지 않는 프로세스를 보조기억장치의 일부(수압) 영역으로 쫓아내고, 당장 실행할 프로세스를 다시 메모리로 적재하는(수압 인) 메모리 관리 기법.
장점: 이 기능을 활용하면 현재 실행하려는 프로세스들이 요구하는 메모리 공간의 총합이 실제 물리 메모리의 크기보다 큰 상황에서도 여러 프로세스를 무리 없이 동시에 실행할 수 있음.

연속 메모리 할당의 3가지 방식 프로세스는 메모리 내의 비어있는 공간에 연속적으로 배치할 때, 빈 공간을 어떻게 찾아 할당할 것인지에 따라 세 가지로 나눌 수 있음.
최초 적합 (First-fit): 메모리 내의 빈 공간을 순서대로 검색하다가 프로세스를 적재할 수 있는 공간을 발견하면 그곳에 바로 할당함. 빈 공간을 찾는 검색 시간이 최소화되어 가장 빠르게 할당할 수 있는 것이 특징임.
최적 적합 (Best-fit): 비어있는 모든 공간을 끝까지 검색한 뒤, 프로세스 크기에 딱 맞는 가장 작은 빈 공간에 할당하는 방식.
최악 적합 (Worst-fit): 비어있는 모든 공간을 끝까지 검색한 뒤, 남은 공간 중 가장 큰 빈 공간에 프로세스를 할당하는 방식.

외부 단편화 (External Fragmentation) : 연속 메모리 할당의 치명적 한계
-> 프로세스들이 수시로 메모리에 적재되고 종료되는 과정이 반복되면 메모리 사이사이에 자잘한 빈 공간들이 생겨남. 이로 인해 공간이 연속적이지 않게 되어 프로세스의 적재가 불가능하게 되어 생기는 메모리 낭비 현상
외부 단편화의 해결 방식 : 메모리 압축, 페이징
메모리 압축 (Compaction): 여기저기 흩어진 빈 공간들을 하나로 밀어 모으는 조각 모음 방식. 하지만 실행 중인 프로세스의 자리를 재배치해야 하므로 오버헤드가 크게 발생하여 프로세스가 제 할 일을 제대로 못 하게 되는 부작용이 있습니다
