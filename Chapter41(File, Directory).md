파일(File)의 정의: 보조기억장치에 저장된 의미 있는 정보들을 한데 모은 논리적 단위
속성(메타데이터): 파일은 실제 데이터 외에도 파일을 설명하기 위한 부가 정보인 속성을 가짐.
속성에는 파일의 크기, 위치, 생성 날짜, 소유자 등이 포함되며, 특히 파일의 유형은 확장자(.exe)를 통해 운영체제에 제공됨.
접근 통제: 응용 프로그램은 임의로 파일에 접근할 수 없으며, 파일을 다루려면 반드시 운영체제의 시스템 호출(System Call)을 통해 요청해야 함.

디렉터리(Directory)의 구조
트리(Tree) 구조: 최상단의 루트 디렉터리(Root Directory)를 시작으로 여러 디렉터리가 계층적으로 구성되는 트리 구조를 사용함.
특별한 파일로서의 디렉터리: 운영체제는 디렉터리와 파일을 크게 구분하지 않고, 디렉터리를 내부에 파일과 하위 디렉터리 정보를 담고 있는 특별한 형태의 파일로 간주함.

디렉터리 엔트리(Directory Entry): 디렉터리 내부는 표(테이블) 형태로 저장되며, 그 표에서의 각 행을 의미함. 여기에는 디렉터리에 포함된 대상의 이름과 보조기억장치 내의 저장 위치를 유추할 수 있는 정보가 들어갑니다.

경로(Path) : 디렉터리를 이용해 특정 파일이나 디렉터리의 위치를 나타내는 정보, 경로가 다르면 동일한 이름의 파일도 존재할 수 있습니다.
절대 경로: 최상위인 루트 디렉터리에서부터 대상까지 이르는 고유한 전체 경로를 의미.
상대 경로: 현재 머물고 있는 디렉터리를 기준으로 대상까지 이르는 경로를 의미.
특수 지시자 : 운영체제와 각종 명령어에서 디렉터리를 탐색할 때 공통으로 쓰이는 기호
점(.): 현재 디렉터리
점점(..): 상위 디렉터리 -> 이를 통해 상위 디렉터리로 거슬러 올라가 접근할 수 있음. (ex/ cd..)
