ALU : 산술 연산과 논리 연산을 수행하기 위해 설계된 디지털 논리 회로의 집합체입니다. 
Ex) 가산기, 보수기, 시프터

 ALU의 입력 정보
- 피연산자 : 레지스터로부터 전달되는 연산 대상 데이터
- 제어 신호 : 제어장치로부터 하달되는 연산 명령

ALU의 출력 정보
- 결과값 : 연산이 완료된 0과 1 형태의 데이터
- 플래그 : 연산 결과에 대한 상태 정보

플래그 : 연산 결과의 부가 정보. 플래그 레지스터에 저장 -> CPU가 다음에 실행할 명령을 결정하는데 참고됨.

부호 플래그 : 결과가 음수인지 양수인지를 나타냄. 연산 결과의 극성을 판별함.
제로 플래그 : 연산 결과가 0인지 아닌지를 나타냄. 비교 연산 시 두 데이터의 일치 여부를 확인함.
캐리 플래그 : 올림수나 빌림수가 발생했을 때 '1'로 설정. 자릿수 초과 및 정밀 연산 처리
오버플로우 플래그 : 연산 결과가 레지스터의 표현 범위를 초과한 것을 나타냄. 연산 오류 검출 및 예외 처리
인터럽트 플래그 : 인터럽트 수용 가능 여부를 결정함. 외부 요청에 대한 응답 제어
슈퍼바이저 플래그 : 현재 실행 모드(커널, 사용자 모드) 표시. 시스템 자원 접근 권한을 제어함.

클럭 신호 : 시스템 내에서 데이터를 주고받도록 박자를 지정하는 역할.
동기화 : 정확한 타이밍에 데이터가 전달되도록 하는 것. -> 클럭 신호가 이를 가능하게 함.
오버클럭킹 : 인위적으로 클럭 주기를 높여 성능을 끌어올리는 행위.

명령어: 명령어 레지스터로부터 전달받은 해석 대상.
플래그: 플래그 레지스터에서 전달된 이전 연산의 상태 정보.
외부 제어 신호: 입출력장치나 주변 장치(키보드, 마우스 등)에서 발생한 인터럽트 및 요청 신호.

레지스터: 레지스터 간 데이터 전송 및 특정 값의 저장을 지시.
ALU: 명령어 해석 결과에 기반하여 수행할 구체적인 산술/논리 연산 유형을 전달.

메모리: 데이터의 읽기 또는 쓰기 동작 제어.
입출력장치: 장치 상태 확인, 데이터 입출력 명령 하달.
