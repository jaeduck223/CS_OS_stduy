프로세스 관리는 수많은 PCB를 시스템 상황에 맞춰 적절한 상태로 전이시키고 자원을 분배하는 고도의 스케줄링 메커니즘입니다.
생성(New) 상태: 프로그램이 메모리에 적재되어 운영체제로부터 PCB를 할당받은 직후의 단계입니다. 커널이 프로세스를 관리할 준비를 마친 상태입니다.
준비(Ready) 상태: 실행에 필요한 모든 준비를 마쳤으나, 아직 CPU 할당을 받지 못해 자신의 차례를 기다리는 상태입니다.
실행(Running) 상태: CPU를 점유하여 실제 명령어를 처리하고 있는 상태입니다. 프로세스는 할당된 시간 동안만 이 상태를 유지합니다.
대기(Blocked) 상태: 실행 중인 프로세스가 입출력 작업과 같은 이벤트를 만나 이를 완료할 때까지 기다리는 상태입니다.
종료(Terminated) 상태: 프로세스 실행이 완료되어 자원을 반납하는 단계입니다. 운영체제는 사용된 메모리를 정리하고 PCB를 폐기합니다.
프로세스의 상태는 특정 하드웨어 인터럽트와 시스템 신호에 의해 유기적으로 변화합니다.
디스패치(Dispatch): 준비 상태의 프로세스가 CPU를 할당받아 실행 상태로 전환되는 핵심 과정입니다.
타이머 인터럽트(Timer Interrupt): 특정 프로세스가 CPU를 독점하는 것을 방지하기 위해, 할당된 시간이 만료되면 발생하는 하드웨어 인터럽트입니다. 프로세스는 다시 준비 상태로 돌아가 공평하게 차례를 기다립니다.
입출력 완료 인터럽트: 대기 상태의 프로세스가 요청한 작업이 완료되었음을 알리는 신호입니다. 이 신호를 받으면 프로세스는 즉시 준비 상태로 복귀하여 재실행을 기다립니다.
운영체제는 프로세스를 개별적인 파편이 아닌, 유기적인 계층 구조로 관리합니다.
부모 프로세스(Parent Process): 새로운 프로세스를 생성한 주체입니다.
자식 프로세스(Child Process): 부모에 의해 생성된 프로세스입니다. 모든 프로세스는 고유한 **PID(Process ID)**를 할당받아 구별됩니다. 특히 자식 프로세스의 PCB에는 부모의 식별자인 **PPID(Parent PID)**가 기록되어, 부모가 자식의 생사 여부를 추적하고 관리할 수 있는 계보를 형성합니다.
프로세스 생성 메커니즘: 복제(fork)와 교체(exec)
fork는 부모 프로세스가 자신의 완벽한 복사본을 생성하는 과정입니다.
자원 상속: 자식 프로세스는 부모가 가지고 있던 메모리 내용, 열린 파일 목록 등을 그대로 물려받습니다.
독립적 실체: 복제된 후에는 부모와 자식이 동일한 코드를 가지지만, 서로 다른 PID와 메모리 공간을 점유하는 완전히 독립적인 개체가 됩니다.
엑섹(exec) 시스템 호출: 정체성의 확립
exec는 프로세스가 자신의 메모리 공간을 새로운 프로그램으로 덮어쓰는 과정입니다.
공간 교체: 호출 시 코드(Code) 영역과 데이터(Data) 영역이 새로운 프로그램의 내용으로 완전히 교체되며, 스택이나 힙 등의 영역은 초기화됩니다.
