메모리 내의 프로그램과 데이터는 고정되어 있지 않고 시시때때로 변화한다! 실행될 때마다 물리적인 데이터의 저장 위치도 달라진다.
-> 동적 적재와 재배치 특성
CPU가 실행하는 모든 명령어는 논리 주소를 기반으로 합니다.

논리 주소: CPU와 실행 중인 프로그램 입장의 주소입니다. 모든 프로그램은 실제 메모리 위치와 상관없이 독립적인 가상 공간을 부여받습니다.
인식하는 주체가 CPU, 프로그램, 컴파일러이며 상대적인 개념입니다.

물리 주소: 메모리 하드웨어(RAM) 입장의 주소입니다. 데이터가 실제로 저장된 전기적 신호의 위치를 의미하며, 전체 시스템에서 중복되지 않는 유일한 주소 값을 가집니다.
인식하는 주체가 RAM입니다. 절대적인 위치의 개념입니다.

MMU(Memory Management Unit, 메모리 관리 장치) : 실제 데이터와 논리 주소 사이의 징검다리 역할을 담당하는 친구. 
핵심 원리 : 물리주소 = 베이스 레지스터 + 논리 주소
베이스 레지스터 : 프로그램의 물리적 시작점
초기 상태 -> 주소 요청 -> 변환 -> 결과 출력

한계 레지스터(Limit Register) : 여러 프로그램이 동시에 실행될 때, 한 프로그램의 오류 또는 악의적인 코드가 다른 프로그램 또는 운영체제를 파괴하는 것을 막는 역할을 담당하는 친구.
한계 레지스터는 해당 프로그램의 논리 주소의 최대 범위를 저장합니다.
논리 주소 발생 -> 경계값 검사 -> 제어 흐름 기반으로 분기
