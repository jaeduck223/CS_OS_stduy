교착 상태 예방 : 교착 상태가 발생하는 4가지 조건 중 하나를 충족하지 못하게 만들어 교착 상태의 발생을 막는 방법.
-> 상호 배제 없애기, 점유와 대기 없애기, 비선점 조건 없애기, 원형 대기 없애기
상호 배제 없애기 : 모든 자원을 공유하게 만드는 방법. -> 말 그대로, 비현실적인 방법.
점유와 대기 없애기 : 특정 프로세스에 자원을 한 번에 모두 할당하거나 아예 할당하지 않는 방식. -> 자원 활용도가 크게 떨어지는 부작용 존재함.
비선점 조건 없애기 : 한 프로세스가 다른 프로세스의 자원을 빼앗을 수 있게 만드는 방식.
원형 대기 없애기 : 모든 자원에 번호를 붙이고 오름차순으로만 자원을 할당하는 방식.

교착 상태 회피 : 교착 상태가 발생하지 않을 선에서만 조심스럽게 자원을 할당하는 방법. -> 자원의 무분별한 할당을 억제하는 기법.
안전 상태 : 안전 순서열이 존재하는 상태
안전 순서열 : 교착 상태 없이 모든 프로세스에 자원을 할당할 수 있는 순서
--> 자원을 할당한 후에도 안전 상태가 유지될 때만 비로소 자원을 할당하여 교착 상태를 회피합니다.

교착 상태 검출 후 회복 : 자원을 요구하는 대로 할당해준 다음, 교착 상태가 발생하면 사후에 조치하는 방법.
선점을 통한 회복 : 교착 상태가 해결될 때까지 한 프로세스씩 자원을 몰아주어 작업을 끝내게 하는 방식.
프로세스 강제 종료를 통한 회복 : 관련된 프로세스를 모두 강제 종료하는 방식.
