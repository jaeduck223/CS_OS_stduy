명령어 파이프라이닝 : 하나의 명령어가 처리되는 과정을 여러 단계로 세분화하여 각 단계를 동시에 실행하는 기법
인출 -> 해석 -> 실행 -> 저장
CPU는 매 클럭마다 서로 다른 명령어의 각 단계를 동시에 수행

파이프라인 위험 : 파이프라이닝의 성능 향상을 저해하고 흐름을 중단시키는 요인
위험 유형 : 데이터 위험, 제어 위험, 구조적 위험
데이터 위험 : 명령어 간의 의존성 때문에 발생하는 현상
제어 위험 : 프로그램 카운터의 갑작스러운 변화로 인해 발생하는 현상
구조적 위험 : 서로 다른 명령어가 동일한 하드웨어 자원을 사용할 떄 서로 간에 충돌하는 현상

슈퍼스칼라 : CPU 내부에 여러 개의 명령어 파이프라인을 두어 한 번에 다수의 명령어를 동시에 처리하는 구조
슈퍼스칼라의 현실 : 실제로는 파이프라인의 개수가 많아질수록 파이프라인 위험의 발생 빈도가 기하급수적으로 증가

비순차적 명령어 처리 : 명령어의 원래 순서에 얽매이지 않고, 실행 가능한 명령어를 먼저 처리하여 파이프라인의 중단을 막는 기법

파이프라이닝은 명령어 처리 단계를 세분화하여 분업의 효율을 달성합니다.
슈퍼스칼라는 파이프라인을 다중화하여 물리적인 확장을 실현합니다.
비순차적 명령어 처리는 의존성을 판단하여 순서를 재배치하는 지능화를 통해 시스템의 흐름 단절을 방어합니다.
