레지스터 : CPU 내부에 위치한 초고속 임시 저장 장치 
역할 : 메모리보다 속도가 월등히 빠르며, CPU가 명령어를 처리하는 과정에서 필요한 데이터와 주소를 아주 짧은 시간 동안 보관하는 역할을 수행
종류 : 프로그램 카운터, 명령어 레지스터, 메모리 주소 레지스터, 메모리 버퍼 레지스터, 플래그 레지스터, 범용 레지스터, 스택 포인터, 베이스 레지스터

프로그램 카운터
: 메모리에서 읽어 들일 다음 명령어의 주소를 저장합니다.
명령어 레지스터 
: 메모리에서 방금 CPU로 가져온, 지금 당장 해석할 명령어를 저장합니다.
메모리 주소 레지스터
: CPU가 주소 버스로 내보낼 메모리의 주소를 저장합니다.
메모리 버퍼 레지스터
: 데이터 버스를 통해 메모리와 주고받는 실제 값을 저장합니다.


주소 준비 및 전달 -> 데이터 인출 -> PC의 즉각적 증가 -> 해석 및 실행

플래그 레지스터
: 연산 결과 및 CPU 상태 정보 (Zero, Carry 등), 이전 연산의 결과가 양수인지, 0인지 등 CPU 상태를 간접적으로 관찰하는 지표
범용 레지스터
: 주소, 데이터, 명령어 등 자유로운 정보. 여러 개가 존재하며, 용도가 정해진 특수 레지스터와 달리 데이터와 주소를 상호 교환하여 저장 가능한 유연성을 가짐

레지스터를 이용한 특수 주소 지정 방식
스택 주소 지정 방식 : 메모리 내부에 미리 지정된 스택 영역을 활용하는 방식

변위 주소 지정 방식
: 명령어의 오퍼랜드 필드에 담긴 값(변위)과 특정 레지스터의 값을 더하여 최종적인 유효 주소를 얻는 방식
