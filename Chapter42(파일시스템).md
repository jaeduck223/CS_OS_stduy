파일 시스템을 사용하기 위한 준비 작업 새 하드디스크나 SSD에 파일 시스템을 구축하여 데이터를 저장하려면 두 가지 선행 작업이 반드시 필요합니다.
파티셔닝(Partitioning): 용량이 큰 저장장치를 하나 이상의 논리적인 단위(파티션)로 구획하는 작업
포맷팅(Formatting): 파티셔닝된 영역에 어떤 파일 시스템을 사용하여 파일을 관리할지 결정하고, 새로운 데이터를 쓸 준비를 하는 과정

보조기억장치에 파일을 할당하는 방법 운영체제는 하드디스크의 작은 저장 단위인 섹터를 여러 개 묶어 블록(Block) 단위로 파일을 읽고 씁니다
연속 할당: 파일을 보조기억장치의 블록에 연속적으로 쭉 이어서 저장하는 단순한 방식! 중간에 빈 공간이 낭비되는 외부 단편화 문제를 야기하기 때문에 오늘날에는 잘 사용되지 않음.
불연속 할당: 파일이 여러 블록에 흩어져 저장되는 기법. 오늘날에 많이 사용하는 기법

연결 할당: 각 블록의 일부에 다음 블록의 주소를 기록하여, 마치 빨랫줄처럼 꼬리를 물고 다음 블록을 가리키는 방식
색인 할당: 파일 데이터가 흩어져 있는 모든 블록 주소들을 색인 블록이라는 단 하나의 블록에 모아서 관리하는 방식입니다

FAT 파일 시스템 (연결 할당 기반): 각 블록에 흩어져 있던 다음 블록의 주소를 한데 모아 FAT(File Allocation Table)라는 표 형태로 관리하는 파일 관리 기법
이 표를 메모리에 올려두면 임의 접근 속도도 매우 빨라지며, 현재까지도 USB 메모리나 SD 카드 등에서 널리 쓰이고 있는 방법임.

유닉스(Unix) 파일 시스템 (색인 할당 기반): 파일의 속성 정보와 데이터가 저장된 15개의 블록 주소를 담고 있는 아이노드(i-node)가 중추적인 역할을 수행
파일 크기가 매우 커서 15개의 주소로 부족할 경우, 주소들을 묶어서 가리키는 단일/이중/삼중 간접 블록 방식을 사용하여 아무리 큰 파일도 유연하게 저장하는 장점이 있음.
모든 핵심 정보를 아이노드가 가지고 있기 때문에, 디렉터리 엔트리에는 단순히 파일 이름과 아이노드 번호만 명시됨.
