프로그램 입출력 : 프로그램 속 명령어로 입출력 장치에 연결된 장치 컨트롤러를 제어하는 방법
CPU가 장치 컨트롤러의 레지스터 값을 직접 읽고 쓰면서 입출력 작업을 수행. 효율이 떨어지는 특징이 있음.
장치 컨트롤러 레지스터 주소 지정 방식 : 메모리 맵 입출력, 고립형 입출럭 등
메모리 맵 입출력 : 메모리 주소 공간과 입출력 장치 주소 공간을 하나의 동일한 주소 공간으로 간주
고립형 입출력 : 메모리용 주소 공간과 입출력 장치용 주소 공간을 완전히 분리하여 할당

인터럽트 기반 입출력 : 입출력 명령을 받은 장치 컨트롤러가 작업을 끝마치면 CPU에게 인터럽트 요청 신호를 보내는 방식. 프로그램 입출력 보다 더 효율적임.
다중 인터럽트 처리 방식 : 순차적 처리, 우선순위 기반 처리

DMA (Direct Memory Access) 입출력 : 입출력 장치가 CPU를 거치지 않고 메모리에 직접 접근하여 데이터를 주고받는 방식. 
CPU가 DMA 컨트롤러에게 입출력 작업을 명령하면, DMA 컨트롤러가 CPU를 대행하여 메모리와 직접 상호작용하며 데이터를 전송함. 모든 작업이 끝나면 CPU에게 인터럽트를 보내 완료를 알림.
특징 : 사이클 스틸링, 입출력 버스, 입출력 전용 프로세스
