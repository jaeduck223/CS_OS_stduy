교착 상태 : 두 개 이상의 프로세스가 각자 자원을 가진 상태에서 서로의 자원을 무작정 기다리기만 하여 그 어떤 프로세스도 끝까지 실행되지 못하고 멈춰버리는 현상
자원 할당 그래프 : 교착 상태가 발생한 상황과 원인을 시각적으로 파악하기 위해 그리는 그래프 (원 = 프로세스, 사각형 = 자원)
교착 상태가 발생한 경우, 이 자원 할당 그래프가 원의 형태를 띔.

교착 상태 발생의 4가지 필수 조건 : 상호 배제, 점유와 대기, 비선점, 원형 대기
상호 배제 : 한 번에 하나의 프로세스만 해당 자원을 사용할 수 있고, 다른 프로세스는 그 자원을 함께 이용할 수 없는 상태
점유와 대기 : 프로세스가 이미 특정 자원을 할당받아 쥐고 있는(점유) 상태에서, 다른 프로세스가 가진 자원을 추가로 기다리고 있는 상태
비선점 : 어떤 프로세스도 다른 프로세스가 사용 중인 자원을 강제로 빼앗을 수 없는 상태
원형 대기 : 프로세스들이 자원을 기다리는 형태가 자원 할당 그래프 상에서 꼬리에 꼬리를 무는 원형(Circular)을 이루고 있는 상태
