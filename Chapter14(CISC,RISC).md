명령어 집합 구조(ISA) : 하드웨어와 소프트웨어 사이의 약속. CPU가 이해하고 실행할 수 있는 명령어들의 총체. ISA는 CPU의 물리적 구조를 결정함.
ISA가 정해지면 그 명령어를 처리하기 위한 레지스터의 개수, 연산 회로의 설계, 명령어 해석 방식 등이 그에 맞춰 최적화되어 정해집니다.
제조사별로 사용하는 ISA가 다르면, 동일한 코드를 컴파일해도 생성되는 기계어가 완전히 달라집니다.

CISC(Complex Instruction Set Computer) : 적은 수의 명령어로 복잡한 기능을 수행하기 위해 고안된 구조. 강력한 명령어 덕분에 컴파일된 코드의 크기가 작은 것이 특징.
하나의 명령어를 실행하는 데 2~4개 이상의 클럭 주기가 소모가 되는 것이 단점임.

RISC (Reduced Instruction Set Computer) : 모든 명령어의 길이를 일정하게 맞춘것이 특징. 1클럭 내 실행을 원칙으로 함.
명령어 내에서 직접 메모리에 접근하는 것을 금지하는 것이 ISA, CISC와의 차이점입니다.

결국에는 CPU들은 성능 최적화를 위해 CISC와 RISC 두 방식의 장점을 영리하게 흡수한 하이브리드 전략을 취하는 것이 바람직함!!
