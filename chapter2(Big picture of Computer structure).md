< 데이터와 명령어 >
- 데이터(Data) : 숫자, 문자, 이미지, 동영상 등 컴퓨터 내부에 저장된 정적인 정보 
                컴퓨터가 0과 1의 이진법을 통해 어떻게 숫자를 표현하고, 문자를 인코딩하여 이해하는지가 데이터 처리의 핵심입니다.
- 명령어(Instruction) : 데이터를 실질적으로 움직이고 처리하게 만드는 동적인 정보

< 컴퓨터의 주요 부품 4가지 - RAM, CPU. ALU, Register 등 >
● 메인 메모리(RAM) : 메모리는 주소(Address) 개념을 통해 방대한 정보 중 필요한 데이터의 위치를 바로 특정하는 역할을 담당
● CPU : 컴퓨터의 두뇌로서 세 가지 내부 요소가 협업합니다.
● ALU(산술논리연산장치) : 실제 수치 연산과 논리 연산을 수행하는 계산기 역할을 담당
● 제어장치(Control Unit) : 명령어를 해석하고, 제어 신호를 내보내 부품들을 관리하는 역할을 담당
● 레지스터(Register) : CPU 내부에 위치한 가장 빠르고 작은 임시 저장소
● 주변 장치(Peripheral Device) : 보조기억장치와 입출력장치를 통칭하는 용어 
-> 보조기억장치 : 메모리의 휘발성을 보완하여 정보를 영구 보관하는 주변 장치 

메모리는 빠르지만 비싸고 전원이 꺼지면 데이터가 사라지는 '휘발성'의 특성이 있음. 
보조기억장치는 느리지만 저렴하고 영구적인 특성이 있음.
-> 따라서 컴퓨터는 보관은 보조기억장치에 하고, 실행은 메모리에서 하는 효율적인 계층 구조를 택하고 있다.

< 메인 보드와 시스템 버스 >
메인보드(Mainboard) : 개별 부품들을 하나의 시스템으로 통합하는 물리적 기반
시스템 버스(System Bus) : 메인보드 안에서 정보가 이동하는 핵심 통로
시스템 버스의 3대 하부 구성 -> 주소 버스, 데이터 버스, 제어 버스
● 주소 버스(Address Bus) : CPU가 정보를 읽거나 쓰기 위해 특정 메모리 번지를 지칭하는 통로
● 데이터 버스(Data Bus) : 명령어와 실제 데이터를 부품 간에 실어 나르는 통로
● 제어 버스(Control Bus) : CPU의 제어장치가 메모리 읽기나 쓰기와 같은 명령을 전달하는 통로

< CPU의 명령어 처리 과정 >
명령어 인출 -> 명령어 해석 -> 데이터 인출 -> 연산 수행 -> 결과 저장
1) 명령어 인출(Fetch): 제어장치가 제어 버스로 신호를, 주소 버스로 메모리 주소를 보냅니다. 메모리는 명령어를 데이터 버스를 통해 CPU의 레지스터로 전달합니다.
2) 명령어 해석(Decode): 제어장치가 가져온 명령어를 해석합니다. 해석 결과를 통해 지시 내용을 파악합니다.
3) 데이터 인출: 제어장치는 다시 메모리에서 데이터를 읽어와 레지스터에 저장합니다.
4) 연산 수행(Execute): 제어장치의 지시에 연산을 수행한 후, 이를 다시 레지스터에 담습니다.
5) 결과 저장(Store): CPU는 다음 명령어를 읽어 해석합니다. 연산 결과를 지시에 따라 제어 버스로는 신호를, 주소 버스로는 메모리 주소를, 데이터 버스에는 결과 값인 데이터를 보내어 메모리에 기록합니다.

- 운영체제(Operating System) : 각 부품이 협업하게 하는 하드웨어의 메커니즘을 가장 효율적으로 관리하고 지휘하는 소프트웨어
