요구 페이징 (Demand Paging) 프로세스의 모든 페이지를 처음부터 메모리에 적재하지 않고, 실행에 필요한 페이지만을 그때그때 메모리에 적재하는 기법입니다
CPU가 특정 페이지에 접근하려 할 때 해당 페이지가 메모리에 없으면(유효 비트가 0일 경우) **페이지 폴트(Page Fault)**가 발생하여 보조기억장치에서 메모리로 페이지를 끌어옵니다

페이지 교체 알고리즘 요구 페이징으로 인해 메모리가 꽉 찼을 때, 어떤 페이지를 보조기억장치로 내보낼지 결정하는 방법입니다
페이지 폴트를 가장 적게 발생시키는 것이 좋은 알고리즘으로 평가받습니다

FIFO (First-In First-Out): 메모리에 가장 먼저 들어온(가장 오래 머문) 페이지를 내쫓는 가장 단순한 방식입니다
하지만 프로그램 내내 사용될 중요한 페이지도 내쫓을 수 있다는 단점이 있습니다

2차 기회 (Second Chance) 알고리즘: FIFO의 단점을 보완하여, 가장 오래된 페이지라도 참조 비트가 1(CPU가 참조한 적 있음)이라면 0으로 초기화한 뒤 기회를 한 번 더 주어 최근에 적재된 것으로 간주하고, 참조 비트가 0일 때만 내쫓는 방식입니다
최적 (Optimal) 알고리즘: 앞으로 가장 오랫동안 사용되지 않을 페이지를 예측하여 교체합니다
가장 낮은 페이지 폴트율을 보장하지만 미래를 예측할 수 없어 실제 구현은 불가능하며, 다른 알고리즘의 성능을 평가하는 이론적 하한선(기준)으로 쓰입니다

LRU (Least Recently Used) 알고리즘: 최적 알고리즘을 모방하여, 과거의 데이터를 바탕으로 지금까지 가장 오랫동안 사용되지 않은 페이지를 교체하는 방식입니다
스레싱 (Thrashing) 프로세스에 할당된 프레임 수가 너무 적을 경우, 실제 프로세스를 실행하는 시간보다 페이지 교체에 더 많은 시간을 쏟게 되어 결국 CPU 이용률과 전체 성능이 급격하게 떨어지는 문제를 말합니다
이를 막기 위해서는 프로세스마다 필요한 최소한의 프레임을 적절히 할당해주어야 합니다
프레임 할당 기법 스레싱을 방지하기 위해 운영체제가 프로세스들에게 프레임을 나누어주는 방법입니다

정적 할당: 프로세스의 실행 과정을 고려하지 않고 고정된 크기만 고려하는 방식입니다
모든 프로세스에 동일한 프레임을 주는 균등 할당과 프로세스 크기에 비례하여 나누어주는 비례 할당이 있습니다

동적 할당: 프로세스의 실제 실행 과정을 관찰하여 유동적으로 할당하는 방식입니다
작업 집합 (Working Set) 모델: 프로세스가 일정 시간 동안 집중적으로 참조한 페이지들의 집합(작업 집합) 크기만큼을 파악하여 최소한 그 개수만큼의 프레임을 할당해 빈번한 페이지 교체를 막는 방식입니다
페이지 폴트 빈도 (Page Fault Frequency) 기반: 페이지 폴트율에 임의의 상한선과 하한선을 정해두고, 상한선을 넘으면 프레임을 더 할당하고 하한선보다 낮으면 프레임을 회수하는 방식입니다
