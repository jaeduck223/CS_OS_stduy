명령어 사이클의 구조: 인출, 실행, 그리고 간접 사이클
명령어 사이클 : CPU가 하나의 명령어를 처리하기 위해 수행하는 기본 단위.
인출 사이클: 실행할 명령어를 메모리에서 CPU 내부 레지스터로 로드하는 단계
실행 사이클: 제어 장치가 인출된 명령어를 해석하고, ALU(산술논리연산장치) 등을 통해 실제 연산을 수행하는 단계
간접 주소 지정 방식 -> 인출한 명령어 내의 주소값이 실제 데이터의 위치가 아닌 '데이터의 주소가 저장된 또 다른 주소'를 가리키게 된다!
이 경우에 한해서는 CPU는 유효 주소를 얻기 위해 메모리에 추가적으로 접근해야 합니다.

인터럽트 : CPU의 정상적인 실행 흐름을 일시적으로 중단시키는 신호
폴링 방식 : 주기적으로 CPU의 상태를 체크하는 방식
동기 인터럽트 (예외): CPU가 명령어를 수행하는 과정에서 직접적으로 직면하는 예기치 못한 상황. 이는 CPU 내부의 논리적 흐름과 동기화되어 발생함.
비동기 인터럽트 : 주로 입출력 장치와 같은 하드웨어에서 발생하는 알림. CPU의 현재 명령어 실행 여부와 관계없이 외부 신호에 의해 발생함.

하드웨어 인터럽트 처리 방식 : 요청 확인 및 사이클 -> Content Backup -> 인터럽트 벡터

인터럽트 처리가 완료되면 CPU는 스택에 백업해 두었던 프로그램 카운터 값을 복구하여 중단되었던 지점부터 다시 실행을 재개합니다.
